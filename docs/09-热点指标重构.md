# 热点指标重构

## 背景

本次重构重新定义热点的两个核心指标：相关性和热度值，简化用户判断热点的认知负担，并优化卡片标签显示。

## 需求确认

1. **保留两个独立指标**：
   - **相关性**：AI 评估内容与关键词的匹配程度（0-100）
   - **热度值**：基于互动量/时效性评估传播热度（0-100）
2. **相关性**：
   - 保存门槛：>= 40 分
   - 分级标签：40-60 轻度关联 / 60-80 高度贴合 / 80-100 直击核心
   - 卡片显示：只显示标签，无前缀后缀，不同颜色
3. **热度值**：
   - 邮箱推送门槛：>= 70 分
   - 分级标签：0-40 无人问津 / 40-70 持续发酵 / 70-90 热议中 / 90-100 刷屏级
   - 卡片显示：只显示标签，无前缀后缀，不同颜色
4. 新增原文语言标识（中文/英文）

### 5. 热点 API 调整

**文件**: `backend/src/routes/hotspots.ts`

#### 相关性分级计算
```typescript
function getRelevanceLevel(score: number | null | undefined): string {
  if (!score) return 'partial';
  if (score >= 80) return 'core';        // 直击核心
  if (score >= 60) return 'relevant';    // 高度贴合
  if (score >= 40) return 'partial';     // 轻度关联
  return 'partial';
}
```

#### 热度分级计算
```typescript
function getHeatLevel(score: number | null | undefined): string {
  if (!score) return 'growing';
  if (score >= 90) return 'viral';       // 刷屏级
  if (score >= 70) return 'hot';         // 热议中
  if (score >= 40) return 'growing';     // 持续发酵
  return 'cold';                          // 无人问津
}
```

```typescript
    await Promise.all([...selectedIds].map(id => hotspotsApi.delete(id)));
    setSelectedIds(new Set());
    setShowBatchConfirm(false);
    loadHotspots();
  } catch (error) {
    console.error('批量删除失败:', error);
    alert('删除失败，请重试');
  } finally {
    setBatchDeleting(false);
  }
};
```
