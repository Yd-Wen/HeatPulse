# 信息源选择与热点优化

## 背景

本次需求包括三个功能：
1. 仪表盘页面新增信息源多选组件（搜狗/B站/X）
2. 热点卡片增加删除功能
3. 调整热点卡片为瀑布流布局

用户已确定方案：
- 信息源默认全选，可取消；无选中时按钮禁用
- 删除需要二次确认
- 使用 CSS Columns 瀑布流布局

### 7. 信息源本地缓存

**问题**：仪表盘页面的信息源选择每次刷新页面后全部恢复为默认选中状态

**解决方案**：使用 localStorage 存储用户的信息源偏好

**修改文件**：
- `frontend/src/pages/Dashboard.tsx`

**方案设计**：

1. **存储键名**：`heatpulse:selected_sources`
2. **默认值**：`['sogou', 'bilibili', 'twitter']`（全选）
3. **存储格式**：JSON 数组

**代码修改**：

1. 添加存储相关常量和函数：
```typescript
// 存储键名
const SOURCES_STORAGE_KEY = 'heatpulse:selected_sources';

// 从 localStorage 读取
const loadSourcesFromStorage = (): SourceType[] => {
  try {
    const stored = localStorage.getItem(SOURCES_STORAGE_KEY);
    if (stored) {
      const parsed = JSON.parse(stored);
      if (Array.isArray(parsed) && parsed.every(s => DEFAULT_SOURCES.includes(s))) {
        return parsed;
      }
    }
  } catch (e) {
    console.error('Failed to load sources from storage:', e);
  }
  return DEFAULT_SOURCES;
};

// 写入 localStorage
const saveSourcesToStorage = (sources: SourceType[]) => {
  try {
    localStorage.setItem(SOURCES_STORAGE_KEY, JSON.stringify(sources));
  } catch (e) {
    console.error('Failed to save sources to storage:', e);
  }
};
```

2. 状态初始化改为从 localStorage 读取：
```typescript
const [selectedSources, setSelectedSources] = useState<SourceType[]>(loadSourcesFromStorage);
```

3. 信息源变更时同步写入（提取为独立函数）：
```typescript
const handleSourceToggle = (source: SourceType) => {
  const newValues = selectedSources.includes(source)
    ? selectedSources.filter((v) => v !== source)
    : [...selectedSources, option.value];
  setSelectedSources(newValues);
  saveSourcesToStorage(newValues);
};
```

4. 立即扫描改用缓存的信息源：
```typescript
const handleQuickScan = async (e: React.FormEvent) => {
  e.preventDefault();
  if (!quickKeyword.trim()) return;

  // 检查是否正在进行扫描
  if (triggering) {
    setAlertMessage('存在进行中的扫描任务');
    setShowAlert(true);
    return;
  }

  // 检查是否选择了信息源
  if (selectedSources.length === 0) {
    setAlertMessage('没有选择信息源，请选择信息源后再进行扫描。');
    setShowAlert(true);
    return;
  }

  try {
    setAdding(true);
    const keyword = await keywordsApi.create(quickKeyword.trim());
    // 使用缓存的信息源
    await scanApi.trigger([keyword.id], selectedSources);
    // ...
  } catch (error) {
    // ...
  } finally {
    setAdding(false);
  }
};
```

**定时扫描说明**：
- 定时扫描保持原有实现，不受本地缓存影响
- 理由：定时扫描是后台自动执行，用户无需感知信息源选择
- 定时扫描会扫描所有可用信息源

**验证方式**：
1. 访问仪表盘页面，取消选择部分信息源，刷新页面，确认选择保持不变
2. 取消所有信息源，点击「立即扫描」，确认弹出提示
3. 选择至少一个信息源，执行立即扫描，确认使用选中的信息源
4. 清除 localStorage 中的 `heatpulse:selected_sources`，刷新页面，确认默认全选
