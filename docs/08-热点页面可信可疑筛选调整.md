# 热点页面可信/可疑筛选调整

## 背景

在实现热点列表排序和筛选功能后，发现"可信/可疑"功能与文档定义的需求存在不一致：

1. **不在需求文档中**：`docs/07-热点列表排序和筛选.md` 未定义此筛选需求
2. **实现方式不一致**：其他筛选通过后端API完成，而"可信/可疑"是前端筛选（仅在已加载数据中过滤）
3. **功能重复**：AI 已通过 `authenticity_score >= 60` 自动过滤低真实性热点

## 决策

**完全移除"可信/可疑"功能**，包括：
- 热点页面的筛选按钮
- 热点卡片上的标签显示

## 修改内容

### 1. 移除筛选按钮

**文件**: `frontend/src/pages/Hotspots.tsx`

移除：
- `filter` 状态
- "全部/可信/可疑"按钮组
- `filteredHotspots` 计算属性
- 未使用的 `Filter` 图标导入

### 2. 移除卡片标签

**文件**: `frontend/src/components/hotspots/HotspotCard.tsx`

移除：
- `Shield`, `ShieldAlert` 图标导入
- 可信/可疑 Badge 显示代码

## 保留的筛选功能

| 筛选类型 | 字段 | 实现方式 |
|---------|------|---------|
| 排序 | created_at/importance/relevance_score/source_type | 后端API |
| 信息来源 | source_type | 后端API |
| 重要性 | importance | 后端API |
| 时间范围 | created_at | 后端API |
| 关键词搜索 | title/content/ai_summary | 后端API |

## 卡片保留的标签

- 信息来源（B站/搜狗/Twitter等）
- 相关性分数（85分等）
- 重要性分类（紧急/高/中/低）
- 匹配的关键词

## 验证方式

1. 访问热点页面
2. 确认无"可信/可疑"筛选按钮
3. 确认卡片上无"可信/可疑"标签
4. 确认其他筛选功能正常
5. 确认相关性分数、重要性标签仍然显示
