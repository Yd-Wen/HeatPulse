# 热点列表排序和筛选功能

## 背景
用户希望在热点列表页面增加排序和筛选功能，并优化热点卡片显示。

## 需求确认（已与用户确认）

### 排序 UI 调整（重要更新）
- **当前方式**：下拉框选择单一排序项 + 降序/升序按钮
- **调整后方式**：
  - 显示所有排序项为可点击按钮：发布时间、重要性、相关性分数、信息来源
  - **仅支持单项排序**（单字段排序）
  - 点击某个排序项时：
    - 如果是当前激活的字段，切换升序/降序
    - 如果是其他字段，切换到该字段并使用默认排序方向
  - 激活的排序按钮显示高亮状态（不同背景色/边框色）
  - 仅激活的按钮显示排序方向箭头（↑ 升序 / ↓ 降序）
  - 默认排序：发布时间降序
- **涉及页面**：热点列表页面（Hotspots.tsx）
- **后端调整**：API 使用 `sort_by` + `sort_order` 单字段排序参数

### 筛选方式
- 信息来源：搜狗/B站/X（source_type）
- 时间范围：今天 / 7天内 / 全部（预设选项）
- 关键词：搜索标题/内容/摘要
- 重要性：按分类（urgent/high/medium/low）

### 重要性分类标准
- urgent: 9-10
- high: 7-8
- medium: 4-6
- low: 1-3

### 热点卡片额外显示
- 添加 importance 分类显示（urgent/high/medium/low 标签）
- 显示匹配的关键词（keyword 字段）
- 关键词标签与来源、相关性分数、重要性标签在同一行显示（footer 区域）

### 邮箱推送调整
- 当前：匹配到有关联邮箱的关键词 OR 设置了全局 NOTIFY_EMAIL 时推送
- 调整后：只推送 importance >= 8 的热点

### 保持不变
- 低相关性热点跳过（authenticity_score < 60 不保存）

### 统一时间字段（重要调整）
- **时间字段统一**：`published_at`（没有则用 `created_at`）
  - 热点卡片显示：已使用 `published_at || created_at`
  - 时间筛选：改为按 `published_at` 筛选
  - 时间排序：改为按 `published_at` 排序
- **时间筛选交互**：
  - 使用预设选项（今天 / 7天内 / 全部）替代手动选择日期
  - 选项值为空表示"全部"
- **时间格式调整**：
  - < 24小时：显示"X分钟前/X小时前"
  - ≥ 24小时且是本年：显示"X月X日"（如"7月10日"）
  - 非本年：显示"XXXX年X月X日"（如"2024年7月10日"）
- **涉及的页面**：
  - 热点卡片（HotspotCard.tsx）
  - 仪表盘（Dashboard.tsx）

### UI 布局
- 筛选条件和排序下拉在同一行，靠近页面顶部
- 使用现有的筛选区域（目前有可信/可疑内容的切换按钮）

## 实施方案

### 1. 时间格式调整
**文件**: `frontend/src/utils/date.ts`

修改 `formatRelativeTime` 函数：
- < 24小时：显示"X分钟前/X小时前"
- ≥ 24小时且是本年：显示"X月X日"（`month: 'short', day: 'numeric'`）
- 非本年：显示"YYYY年M月D日"（`year: 'numeric', month: 'short', day: 'numeric'`）

### 2. 后端 API 单字段排序支持
**文件**: `backend/src/routes/hotspots.ts`

- 使用 `sort_by` + `sort_order` 参数进行单字段排序
- 保持向后兼容：也支持 `sort` 多字段排序参数

### 3. 前端 API 类型调整
**文件**: `frontend/src/types/index.ts`

- `HotspotsQueryParams` 类型已支持单字段排序（`sort_by` + `sort_order`）
- 无需额外修改类型定义

### 4. 前端排序 UI 调整
**文件**: `frontend/src/pages/Hotspots.tsx`

- 移除单一下拉框选择 + 降序/升序按钮的方式
- 改为显示所有排序项为按钮组：发布时间、重要性、相关性分数、信息来源
- 状态管理改为单字段排序：`activeSortField` + `sortOrder`
- 点击按钮时：
  - 如果点击的是当前激活字段：切换升序/降序
  - 如果点击的是其他字段：切换到该字段，使用默认方向
- 激活的按钮显示高亮状态（不同背景色/边框色）
- 仅激活的按钮显示排序方向箭头（↑ 升序 / ↓ 降序）
- 请求 API 时传递 `sort_by` + `sort_order` 参数

### 5. 后端 API 筛选字段调整
**文件**: `backend/src/routes/hotspots.ts`

- 筛选字段从 `created_at` 改为 `published_at`（优先）没有则用 `created_at`
- 由于 SQLite 不能直接用 IFNULL，需要在代码中处理逻辑
- 排序字段从 `created_at` 改为 `published_at`

### 2. 邮箱推送调整
**文件**: `backend/src/services/scanner/index.ts`

- 邮件发送条件增加 importance >= 8 检查

### 3. 前端 API 客户端
**文件**: `frontend/src/api/client.ts`

扩展 `HotspotsQueryParams` 类型和 Hotspot 类型

### 4. 前端热点卡片组件
**文件**: `frontend/src/components/hotspots/HotspotCard.tsx`

- 添加 importance 分类标签
- 显示匹配的关键词

### 5. 前端热点页面
**文件**: `frontend/src/pages/Hotspots.tsx`

- 将日期 input 替换为预设选项下拉框：今天 / 7天内 / 全部
- 当选择"今天"时：start_date = 今天 00:00:00，end_date = 今天 23:59:59
- 当选择"7天内"时：start_date = 7天前 00:00:00，end_date = 今天 23:59:59
- 当选择"全部"时：start_date 和 end_date 为空

2. 启动前端: `cd frontend && npm run dev`
3. 访问热点页面测试：
   - 观察卡片是否显示 importance 分类和关键词

   - 选择重要性分类，观察结果
   - 组合使用多个筛选条件
