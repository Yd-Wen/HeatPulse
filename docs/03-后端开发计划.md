# HeatPulse 后端开发计划

## 阶段 1：项目初始化与基础架构

### 1.1 初始化 Express + TypeScript 后端项目
- 创建 backend 目录
- 初始化 package.json，安装依赖：
  - `npm install express@5 prisma @prisma/client ws node-cron nodemailer axios cheerio dotenv cors`
  - 开发依赖: `npm install -D typescript @types/express @types/node @types/ws @types/node-cron @types/nodemailer ts-node nodemon prisma`
- 配置 TypeScript (tsconfig.json)
- 创建 src 目录结构

### 1.2 创建数据库层 (Prisma)
- 初始化 Prisma: `npx prisma init --datasource-provider sqlite`
- 创建 Prisma Schema (prisma/schema.prisma)
- 定义数据模型: Keyword, Hotspot, Notification
- 运行迁移: `npx prisma migrate dev --name init`
- 创建 Prisma Client 单例 (src/utils/prisma.ts)

### 1.3 创建 Express 应用入口
- 配置 app.ts (中间件、路由、错误处理)
- 创建 server.ts (服务入口)

## 阶段 2：API 路由实现

### 2.1 关键词管理 API
- `GET /api/keywords` - 获取所有关键词
- `POST /api/keywords` - 创建关键词
- `PATCH /api/keywords/:id` - 更新关键词
- `DELETE /api/keywords/:id` - 删除关键词

### 2.2 热点管理 API
- `GET /api/hotspots` - 获取热点列表（支持分页、筛选）
- `GET /api/hotspots/:id` - 获取热点详情
- `POST /api/hotspots/:id/read` - 标记已读

### 2.3 扫描控制 API
- `POST /api/scan/trigger` - 手动触发扫描
- `GET /api/scan/status` - 获取扫描状态

### 2.4 统计 API
- `GET /api/stats` - 获取统计数据

## 阶段 3：核心业务逻辑

### 3.1 数据源服务 (多源获取)
创建 `services/scanner/sources/` 目录，实现多源数据获取：

#### 3.1.1 网页搜索爬虫 (sources/web.ts)
- 使用 cheerio + axios 实现网页爬取
- 目标网站（均无需 API Key）：
  - **微博热搜** (weibo.com) - https://weibo.com/ajax/side/hotSearch
  - **知乎热榜** (zhihu.com) - https://www.zhihu.com/api/v3/feed/topstory/hot-lists/total
  - **百度热榜** (top.baidu.com) - https://top.baidu.com/board?tab=realtime
- 实现内容提取：
  - 标题
  - 摘要/描述
  - 热度数值
  - 来源链接
- 频率控制：每个源间隔 3-5 秒
- 反爬应对：随机 User-Agent，添加 Referer 请求头

#### 3.1.2 B站公开 API (sources/bilibili.ts)
- 使用 B站公开 API，无需 API Key
- 实现功能：
  - 获取热门视频：`GET /x/web-interface/popular`
  - 搜索视频：`GET /x/web-interface/search/type`
  - 获取用户信息：`GET /x/web-interface/view/info` (通过 av号)
  - 获取用户空间：`GET /x/space/acc/info` (通过 mid)
- 频率限制：每 IP 每接口 1 次/秒
- **特殊处理**：当关键词是 B站账号时（如 @用户名），调用用户信息接口获取该账号数据
- 环境变量：无需配置

#### 3.1.3 账号识别逻辑
- 当关键词以 `@` 开头时，识别为账号查询
- 各平台识别规则：
  - 微博：`@用户名` → 调用微博用户搜索 API
  - B站：`@用户名` → 调用 B站用户信息 API
  - 知乎：`@用户名` → 搜索用户
- 获取账号信息：
  - 头像
  - 简介
  - 粉丝数
  - 最新动态/内容

#### 3.1.4 Twitter API 集成 (sources/twitter.ts)
- 对接 twitterapi.io API
- 实现功能：
  - 获取趋势话题 (Trends)
  - 搜索关键词相关推文
- API 端点：
  - `GET /v2/trends/by-place` - 获取指定地区的趋势
  - `GET /v2/tweets/search` - 搜索推文
- 频率控制：API 调用间隔 10-15 秒
- 环境变量：`TWITTER_API_KEY`

#### 3.1.5 统一数据源接口
创建 `sources/index.ts` 统一入口：
```typescript
interface SourceResult {
  title: string;
  summary: string;
  url: string;
  source: string;
  heat: number;
  timestamp: Date;
  isAccount?: boolean;  // 是否为账号信息
  accountInfo?: {
    avatar?: string;
    followers?: number;
    description?: string;
  };
}

// 判断是否为账号查询
function isAccountQuery(keyword: string): boolean

// 获取所有源的数据
async function fetchAllSources(keywords: string[]): Promise<SourceResult[]>
```

### 3.2 扫描服务 (Scanner Service)
- 创建扫描主逻辑 (services/scanner/index.ts)
- 实现扫描流程：
  1. 获取所有关键词
  2. 并行从多个数据源获取数据
  3. 数据清洗和去重
  4. 调用 AI 分析
  5. 存储热点

### 3.3 AI 集成 (OpenRouter + DeepSeek)
- 创建 OpenRouter 封装 (services/ai/openrouter.ts)
- 创建 Prompt 模板 (services/ai/prompts.ts)
- 实现热点分析逻辑：
  - 真实性判断
  - 重要性评分 (0-100)
  - 摘要生成 (50-100字)
  - 标签提取

### 3.4 WebSocket 服务
- 创建 WebSocket 服务器 (services/websocket.ts)
- 实现广播功能：
  - 新热点推送
  - 扫描状态推送
- 心跳机制：每 30 秒 ping/pong

## 验证方式

1. **数据源验证**: 分别测试各数据源是否能获取数据
2. **数据库验证**: `npx prisma studio` 查看数据表结构正确
3. **API 验证**: 使用 curl 测试所有 API 端点
4. **前端集成验证**: 启动后端后，前端能正常请求数据
5. **WebSocket 验证**: 前端能收到实时推送
6. **扫描功能验证**: 手动触发扫描，检查热点是否生成

## 注意事项

- 使用 Prisma ORM 替代 better-sqlite3 原生查询
- 爬虫实现需要注意频率控制，避免被封（每次请求间隔 3-5 秒）
- B站 API 有严格的频率限制（1次/秒），需严格遵守
- 账号查询功能：关键词以 `@` 开头时自动识别为账号查询
- AI 调用需要处理错误和重试逻辑（最多 3 次重试）
- WebSocket 需要处理连接断开和重连
- Prisma 需要运行 migrate 后才能使用
- Twitter API 有调用频率限制，需遵守 twitterapi.io 的规则
