## API 端点汇总

```
GET    /api/keywords              # 获取所有关键词
POST   /api/keywords              # 创建关键词
DELETE /api/keywords/:id          # 删除关键词
PATCH  /api/keywords/:id          # 更新关键词

GET    /api/hotspots              # 获取热点列表（支持分页、筛选）
GET    /api/hotspots/:id          # 获取热点详情
POST   /api/hotspots/:id/read     # 标记已读

POST   /api/scan/trigger          # 手动触发扫描
GET    /api/scan/status           # 获取扫描状态

GET    /api/stats                 # 获取统计数据
```

## WebSocket 消息类型

```typescript
// 新热点推送
{ type: 'NEW_HOTSPOT', data: Hotspot, timestamp: string }

// 扫描开始
{ type: 'SCAN_START', data: ScanStatus, timestamp: string }

// 扫描完成
{ type: 'SCAN_COMPLETE', data: ScanStatus, timestamp: string }

// 心跳
{ type: 'PING' | 'PONG', timestamp: string }
```

## 验证方式

1. **数据源验证**: 分别测试各数据源是否能获取数据
2. **数据库验证**: `npx prisma studio` 查看数据表结构正确
3. **API 验证**: 使用 curl 测试所有 API 端点
4. **前端集成验证**: 启动后端后，前端能正常请求数据
5. **WebSocket 验证**: 前端能收到实时推送
6. **扫描功能验证**: 手动触发扫描，检查热点是否生成

## 注意事项

- 使用 Prisma ORM 替代原生查询
- 爬虫实现需要注意频率控制，避免被封（每次请求间隔 3-5 秒）
- B站 API 有严格的频率限制（1次/秒），需严格遵守
- 账号查询功能：关键词以 `@` 开头时自动识别为账号查询
- AI 调用需要处理错误和重试逻辑（最多 3 次重试）
- WebSocket 需要处理连接断开和重连
- Prisma 需要运行 migrate 后才能使用
- Twitter API 有调用频率限制，需遵守 twitterapi.io 的规则
