# 优化信息源获取

## 问题描述

当前扫描服务存在信息源获取失败的问题：

```
[Scanner] Scanning 1 keywords
[Bilibili] Failed to search videos [412]: Request failed with status code 412
[Bilibili] Response: {
  "code": -412,
  "message": "request was banned",
  "ttl": 1
}
[Twitter] Failed to fetch tweets [404]: Request failed with status code 404
[Twitter] Response: {
  "detail": "Not Found"
}
```

## 问题诊断

### 1. Bilibili 返回 412 错误

**原因**: B站有反爬虫机制，检测到异常请求时会返回 412 (请求被禁止)

**当前代码问题** (`backend/src/services/scanner/sources/bilibili.ts`):
- 缺少必要的 Cookie (buvid3)
- User-Agent 可能被识别为机器人
- 请求频率过高

### 2. Twitter 返回 404 错误

**原因**:
- Twitter API 端点可能已变更
- twitterapi.io 服务可能不稳定
- API Key 可能无效或已过期

## 参考项目

参考 [yupi-hot-monitor](https://github.com/liyupi/yupi-hot-monitor) 的实现方案：

| 数据源 | 类型 | 需要API Key | 说明 |
|-------|------|-------------|------|
| Twitter (twitterapi.io) | API | 可选 | 官方API的第三方封装 |
| Bing/Google | 搜索引擎 | 否 | 网页爬虫 |
| HackerNews | API | 否 | 官方API |
| 搜狗 | 搜索引擎 | 否 | 中文搜索 |
| B站 | API | 否 | 公共接口 |
| 微博 | 热搜API | 否 | 公共接口 |

## 解决方案

修复 B站和 Twitter API，然后添加微博、搜狗等中国免费数据源。

### 步骤 1: 修复 Bilibili 数据源

**文件**: `backend/src/services/scanner/sources/bilibili.ts`

修改内容:
- 添加 `buvid3` Cookie (UUID + "infoc")
- 完善请求头 (Accept-Language, Connection 等)
- 添加请求延迟避免 412

参考 yupi-hot-monitor 实现:
```python
def _bili_headers():
    buvid3 = f"{uuid.uuid4()}infoc"
    return {
        "User-Agent": rand_ua(),
        "Referer": "https://search.bilibili.com/",
        "Accept": "application/json",
        "Cookie": f"buvid3={buvid3}",
    }
```
