# HeatPulse 需求

## 项目背景

作为一个 AI 编程学习者，需要第一时间获取 AI 领域的热点（如大模型更新），不能依赖人工手动搜索，而是利用工具自动发现指定的热点变化，或者发现最新的热点，并且及时发送通知。

## 功能需求

### 1. 关键词监控
- 用户手动输入要监控的关键词
- 当关键词相关内容出现时，第一时间发送通知
- AI 识别真实内容 vs 假冒内容

### 2. 热点自动发现
- 每隔 30 分钟自动扫描指定领域（如"AI 编程"）
- 聚合多源信息，发现最新热点
- AI 分析热点真实性和重要性

### 3. 通知机制
- 浏览器实时推送（WebSocket）
- 邮件通知

### 4. 产品形态
- 响应式兼容的 Web 页面
- 封装成 Agent Skills 技能，供其他 AI Agent 使用

## 非功能需求

### 技术选型
- **前端**: React 18 + Vite + TypeScript + Tailwind CSS
- **后端**: Express 5 + TypeScript + SQLite
- **AI**: DeepSeek via OpenRouter
- **实时通信**: WebSocket (ws)
- **邮件**: Nodemailer
- **定时任务**: node-cron

### 数据源
1. **网页搜索爬虫** - 直接通过网页搜索网址爬虫获取（注意频率控制）
2. **Twitter/X API** - 从 https://twitterapi.io/ 获取
3. **多源聚合** - 避免单一信息源

### 设计要求
- 前端页面足够独特，非千篇一律
- 深色主题 + 霓虹渐变（脉冲概念）
- 响应式布局（移动端/桌面端）

### 开发顺序
1. 先完成网页版，确保功能正常使用
2. 再开发 Agent Skills

## 环境准备

- VS Code
- Claude Code
- 大模型 API Key (OpenRouter)
- Node.js 18+

## 验收标准

- [ ] 可以添加/删除监控关键词
- [ ] 每 30 分钟自动扫描热点
- [ ] AI 正确识别热点真实性
- [ ] 新热点出现时浏览器实时推送
- [ ] 新热点出现时发送邮件通知
- [ ] 响应式页面在移动端正常显示
- [ ] Agent Skills 可以被其他 AI 调用
